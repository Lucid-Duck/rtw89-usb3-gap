# D-Link DWA-X1850 Rev A1

## Identity

| Field | Value |
|---|---|
| Marketing name | D-Link DWA-X1850, Rev A1 |
| VID:PID | `2001:3321` |
| Chip family | RTL8852AU family (binds to `rtw89_8852au_git`) |
| Generation | AX (Wi-Fi 6) |
| Firmware | `rtw8852a_fw-1.bin`, version 0.13.36.2 (52acc807) |
| Chip info from firmware | CID: 0, CV: 2, AID: 0, ACV: 0, RFE: 1 |

## Distinguishing features from the B1

Same driver module and same firmware as the B1, but a genuinely different hardware revision:

1. **USB 2 descriptor string**: A1 reports "802.11ac WLAN Adapter" on the USB 2 side (stale Wi-Fi 5 marketing string). Switches to "802.11ax WLAN Adapter" after re-enumeration on USB 3. B1 reports "802.11ax" on both sides.
2. **Installer shim**: A1 exposes a `0bda:1a2b "DISK"` virtual CD-ROM device for approximately 3 seconds before presenting the wireless device. B1 does not.
3. **Switch-mode register value**: A1 writes `0xb0f10` to register 0xc4. B1 writes `0xb0f00`. One-bit difference in the value. This matches the AX4L behavior, suggesting the A1 and AX4L share a register-value variant that the B1 does not.

## Identification pass results

### Hub topology (J5 Create powered USB 3.0 hub, 2026-04-11)

- Initial enumeration path: `3-1.3` (USB 2 high-speed via hub)
- Final path: `2-1.3` (USB 3 SuperSpeed via hub)
- Negotiated speed: **5000 Mbit/s SuperSpeed**
- Full dmesg: `evidence/dmesg/framework-morrownr-git/dwa-x1850-a1-hub-plug1-2026-04-11.log`

### Direct slot topology (Framework bottom-right USB-A, 2026-04-11)

- Initial enumeration path: `3-3` (USB 2 high-speed companion root)
- Final path: `2-2` (USB 3 SuperSpeed root)
- Negotiated speed: **5000 Mbit/s SuperSpeed**
- Full dmesg: `evidence/dmesg/framework-morrownr-git/dwa-x1850-a1-direct-plug1-2026-04-11.log`

## Topology invariance

Both hub and direct-slot runs produced identical switch-mode signatures. The switch-mode code is not sensitive to USB topology.
